## 使用建议

1. **开发前必读**：创建新功能前，先浏览对应的规则文件
2. **参考文件**：每个文件都包含具体的参考文件位置和示例代码
3. **复用检查**：在 `components.md` 中检查是否已有可用组件，避免重复开发
4. **规范遵循**：严格按照规范编写，保持代码风格一致
5. **类型安全**：优先查看 `ts-define.md`，确保类型定义规范

## 与 CLAUDE.md 的关系

这些规则文件是从 CLAUDE.md 中拆分而来的细分文档。原始的 CLAUDE.md 仍然保留，包含了项目的整体概览、业务模块、开发命令等信息。

- **查看具体规范** → 参考 `.claude/rules/` 中的细分文件
